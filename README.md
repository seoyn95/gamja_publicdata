# Gamja Public Data

기업 리뷰 및 채용공고 기반 이직 사유 매칭 시스템입니다.
2025.03 - 2025.06

## 주요 기능
- 잡플래닛 리뷰 크롤링
- 채용 공고 크롤링
- OpenAI 기반 임베딩
- 이직 사유 기반 기업 추천

## 폴더 구조
- `기업 임베딩 GPT/`: GPT 기반 임베딩 처리 코드
- `잡플래닛 리뷰/`: 리뷰 스크래핑 및 전처리
- `잡플래닛 채용공고/`: 채용정보 수집
- 기타 `.csv`, `.xlsx`: 데이터 결과 파일

**플랫폼 개요**

---

본 프로젝트는 이직 및 재취업을 준비하는 사용자들이 자신의 경력, 선호, 이직 사유에 기반하여 적합한 기업 및 직무를 효과적으로 탐색할 수 있도록 지원하는 맞춤형 진단·추천 플랫폼 구축을 목표로 한다. 특히, 조직문화 적합성, 산업 특성, 이직률 등 정성적·정량적 데이터를 종합 분석하여, 현실적이고 개인화된 추천을 제공하는 것이 핵심 기능이다.

---

**핵심 기능 및 데이터 흐름**

---

∙ 잡플래닛 리뷰 크롤링

∙ 채용 공고 크롤링

∙ OpenAI 기반 임베딩

∙ 이직 사유 기반 기업 추천

∙이직 사유 입력 → 산업/지역 필터링 → 기업 리뷰 임베딩 → 유사도 분석 → 추천/제거 → 채용공고 출력

---

**데이터 명세서 및 로직**

---

본 프로그램은 사용자의 입력값(산업, 직무, 지역, 이직 사유 코드)을 기반으로 다음과 같은 절차를 통해 기업과 직무를 추천한다.

---

1. **산업/지역 필터링 (및 채용공고 매칭)**

---

사용자가 입력한 산업코드 및 지역코드를 기준으로, 사전에 수집된 채용공고 데이터에서 해당 조건을 만족하는 기업을 1차 필터링한다.

| 항목 | 설명 |
| --- | --- |
| 입력 데이터 | 사용자 입력: 산업코드, 지역코드, 직무코드, 이직 사유 코드 |
| 산업 필터링 | 업종코드.xlsx로 산업 분류 (앞 2자리 코드 매칭) |
| 지역 필터링 | 업종코드.xlsx로 산업 분류 (앞 2자리 코드 매칭) |
| 출력 | 필터링된 채용 공고 (추천_채용공고.csv) |

---

1. **이직 사유 임베딩 및 의미 분석**

---

이직 사유는 구조화된 텍스트가 아니기 때문에, 사유별 예시 문장을 OpenAI의 text-embedding-3-small 모델을 활용하여 벡터화하고, 의미 기반 유사도 분석을 가능케 했다.

| 항목 | 설명 |
| --- | --- |
| 기능 | JobPlanet에서 리뷰 자동 크롤링 (로그인 자동화 포함) |
| 수집 대상 | 리뷰 요약, 장점, 단점, 경영진 의견 |
| 전처리 | 텍스트에서 중복 제거, 불법 문자 제거, 구간별 정리 |
| 출력 | 기업별 .xlsx 파일 (예: 잡플래닛 리뷰_삼성전자.xlsx) |

---

1. **기업 리뷰 임베딩 및 조직문화 분석**

---

기업 리뷰는 JobPlanet에서 자동 크롤링하여 확보하였고, 리뷰의 ‘장점’, ‘단점’, ‘경영진 의견’ 등을 긍정/부정으로 분류한 후 임베딩하였다. 이 과정에서 국내 SBERT(KoSimCSE) 모델과 OpenAI 모델을 병행 적용하였다.

| 항목 | 설명 |
| --- | --- |
| 사용 데이터 | 이직 사유 예문.CSV (사유별 예문 다수 포함) |
| 임베딩 도구 | OpenAI text-embedding-3-small |
| 출력 | sentence_embeddings_openai.npy 및 임베딩_결과_openai.csv |
| 특징 | 사유별 문장을 벡터화하여 후속 유사도 분석에 활용 |

---

1. **유사도 기반 기업 추천/제거**

---

**(1) 기업 리뷰 임베딩 (기반 데이터 생성)**

유사도의 이직 사유 벡터와 각 기업의 부정 리뷰 임베딩 간 코사인 유사도를 비교하여, 유사도가 높은 기업을 선별하고, 불일치도가 높은 기업은 필터링함으로써 조직문화 기반 맞춤 추천을 실행하였다.

| 항목 | 설명 |
| --- | --- |
| 기능 | 스크래핑된 리뷰에서 긍정/부정 문장 추출 후 임베딩 |
| 사용 모델 | ∙KoSimCSE 
∙OpenAI 임베딩 |
| 출력 | {기업명}_positive.npy, {기업명}_negative.npy |

(**2) 기업–이직사유 유사도 분석 (이직자 맞춤 유사 기업 추출)**

| 항목 | 설명 |
| --- | --- |
| 비교 대상 | 사용자의 이직 사유 임베딩 vs. 기업 리뷰 임베딩 (negative) |
| 유사도 계산 | cosine similarity |
| 출력 | 기업별 top 5 이직 사유 유사도 리스트 (기업_이직사유_추천.csv) |
| 정제 방식 | 평균 유사도 계산 후 유사도 높은 사유를 포함한 기업만 추출 |

(**3)불일치(반대) 기업 필터링 (부정적 기업 제거)**

| 항목 | 설명 |
| --- | --- |
| 사용목적 | 이직 사유와 정반대 분위기를 가진 기업 제거 |
| 데이터 | 반대기업특징_리스트.CSV (이직자의 기피 특성 예시) |
| 임베딩 방식 | ∙반대 임배딩.PY → KoSimCSE 
∙리버스 임베딩 KoSimCSE.py → OpenAI 기반 |

---

1. **최종 채용공고 매칭 및 출력**

---

본 단계는 사용자에게 최종적으로 추천할 채용 공고 목록을 확정하고 제시하는 과정입니다. 앞선 단계에서 사용자 입력값(산업, 지역)을 기반으로 1차 필터링을 거친 채용 공고 데이터가 사용한다.

(**1) 데이터 준비 및 활용**

1차 필터링을 통해 확보된 채용 공고 데이터는 추천 대상의 초기 목록으로 활용된다. 여기에 더하여, 이직 사유와 기업 간의 연관성 정보를 담고 있는 기업_이직사유_추천.csv 파일을 활용한다. 이 파일로부터 사용자가 입력한 이직 사유 코드에 해당하는 이직 사유 텍스트와 연관성이 높다고 분석된 기업 리스트를 도출한다.

| 항목 | 설명 |
| --- | --- |
| 목적 | 최종 추천 목록 필터링 프로세스에 필요한 초기 채용 공고 데이터와 이직 사유 기반 제외 대상 기업 목록을 준비 |
| 입력 데이터 | ∙1차 필터링 채용 공고 데이터
∙기업_이직사유_추천 데이터
∙사용자 입력 이직 사유 코드 |
| 처리 로직 | ∙사용자 입력 이직 사유 코드를 사용하여 코드에 해당하는 이직 사유 텍스트를 내부 매핑 정보에서 확인
∙기업_이직사유_추천 데이터 내에서 확인된 이직 사유 텍스트와 관련이 있으며, 사전에 정의된 기준(유사도 임계값)을 만족하는 기업들을 식별 |
| 산출 데이터 | 제외 대상 기업 목록 |

(**2) 추천 목록 확정 (제외 로직 적용)**

최종 추천 목록을 확정하기 위해 1차 필터링된 채용 공고 목록에서 특정 기업들을 제외하는 로직을 적용한다. 구체적으로, 사용자의 이직 사유와 연관성이 높은 것으로 파악된 기업들의 채용 공고를 초기 추천 목록에서 제외시키는 과정이다. 이 과정을 통해 사용자의 특정 이직 사유와 부정적인 관련성이 있는 기업들을 사전에 필터링하여 보다 적합한 추천 결과를 제공할 수 있다.

| 항목 | 설명 |
| --- | --- |
| 목적 | 제외 대상 기업 목록을 활용하여 1차 필터링된 채용 공고 데이터에서 최종 추천 목록을 확정 |
| 입력 데이터 | ∙1차 필터링 채용 공고 데이터
∙제외 대상 기업 목록 |
| 처리 로직 | ∙1차 필터링 채용 공고 데이터 내의 각 채용 공고 정보를 순회
∙해당 채용 공고의 회사명이 제외 대상 기업 목록에 포함되어 있는지 확인
∙제외 대상 기업 목록에 포함되지 않는 채용 공고 정보만을 선택하여 새로운 데이터 집합을 구성 |
| 산출 데이터 | 최종 추천 채용 공고 목록: 사용자 입력 및 이직 사유 기반 필터링이 완료된 최종 추천 대상 채용 공고 정보의 집합 (filtered_df) |

**(3) 결과 제시 및 저장**

필터링 과정이 완료되면, 추천 대상에서 제외된 기업의 수와 최종 추천 목록에 남은 기업의 수를 사용자에게 제시하여 필터링 결과를 직관적으로 확인할 수 있도록 한다. 또한 최종 필터링된 추천 데이터 중 핵심 정보(회사명, 직무, 경력 등)의 일부를 사용자에게 즉시 제공해 추천 내용을 간략하게 파악할 수 있게 지원한다. 더 나아가 최종 추천 목록 전체는 별도의 파일(최종 추천 기업.csv)로 저장되어 사용자가 추후에도 해당 정보를 활용할 수 있도록 한다.

| 항목 | 설명 |
| --- | --- |
| 목적 | 최종 처리 결과를 사용자에게 제시하고 영구적으로 저장 |
| 입력 데이터 | ∙1차 필터링 채용 공고 데이터(비교를 통한 통계 산출용)
∙최종 추천 채용 공고 목록 |
| 처리 로직 | ∙1차 필터링 채용 공고 데이터의 개수와 최종 추천 채용 공고 목록의 개수를 비교하여 제외된 기업 수를 계산
∙최종 추천 채용 공고 목록의 개수를 통해 최종 남은 기업 수를 확인
∙최종 추천 채용 공고 목록에서 회사명, 직무, 경력 등 핵심 정보를 선별
∙최종 추천 채용 공고 목록 전체를 .csv 형식의 파일(최종 추천 기업.csv)로 저장 |
| 산출 데이터 | ∙화면 출력 통계: 제외된 기업 수, 최종 남은 기업 수 (사용자 인터페이스에 제시)
∙화면 출력 샘플 데이터: 최종 추천 목록의 일부 정보 (회사명, 직무, 경력 등)
∙최종 추천 파일: 최종 추천 기업.csv (사용자 활용을 위한 영구 저장 파일) |

위 과정을 통해 추출된 적합 기업에 대해 채용공고를 정리 및 출력하며, 사용자는 자신의 사유와 상황에 적합한 기업 정보를 확인할 수 있다.
